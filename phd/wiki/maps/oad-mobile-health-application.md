# OAD Mobile Health Application

The [[phd/raw/OAD/OAD]] (Outil d'Aide au Diagnostic), formally managed as the [[phd/wiki/entities/oad|OAD]] project ([[phd/wiki/sources/notes2026oad|OAD — Project Hub]]), is a [[phd/wiki/concepts/react-native|React Native]] Android tablet application. Also referred to as the [[OAD Mobile Health Application]], it is designed to assist healthcare workers—specifically 'Agents de Santé' and Community Health Agents—in the Central African Republic and other resource-limited sub-Saharan settings. Functioning as a [[phd/wiki/concepts/syndromic-approach|syndromic diagnostic aid tool]] grounded in a clinical [[Syndromic Approach]], it enables the diagnosis and treatment of non-malarial febrile illnesses without laboratory confirmation. As outlined in the [[phd/wiki/sources/notes2025oadhandoff|OAD(handoff) - product requirements document]], this diagnostic [[phd/wiki/concepts/syndromic-approach|Syndromic Approach]] aims to reduce unnecessary prescriptions, integrate AI decision support, and demonstrate economic viability by replacing legacy Excel-based systems. The initial release requires successfully porting all existing Excel-based diagnostic functions to the application. Early technical scoping ([[phd/wiki/sources/notes2025oadhandoff|OAD(handoff) - React Native ou ODK]]) compared the [[phd/wiki/concepts/react-native|React Native]] framework against [[phd/wiki/concepts/open-data-kit|Open Data Kit]] (commonly referred to as [[phd/wiki/concepts/open-data-kit|ODK]]). While the [[phd/wiki/concepts/open-data-kit|ODK]] route offered a fast one-month build with built-in offline synchronization, it yielded a static UI and lacked dynamic capabilities without a paid tier for entity management. Consequently, the chosen [[phd/wiki/concepts/react-native|React Native]] path prioritized a custom UX with the interactive, guided design necessary for fast-paced medical environments, necessitating a four-month full-stack engineering investment across app, server, infrastructure, and security development. To guarantee uninterrupted functionality in remote areas, the [[phd/wiki/entities/oad|OAD]] mobile application utilizes an [[Offline-First Architecture]]. This [[phd/wiki/concepts/offline-first-architecture|offline-first architecture]] ensures operability by reading and writing to a local device database and syncing with distant servers when connectivity allows, effectively fulfilling the need for an [[phd/wiki/concepts/offline-first-architecture|Offline-First Architecture]] tailored for the field. In early 2026, the tablet-based application built with [[React Native]] underwent a significant refactor to strictly enforce this [[Offline-First Architecture]] ([[phd/wiki/sources/notes2026oadlog|OAD(log) - Tablet Database]]). The data layer was transitioned to a Model-Repository-Service pattern, relying on [[RxDB]] for persistent local data storage and observable subscriptions, while restricting [[Zustand]] exclusively to UI state management and data pointers. The project is spearheaded by Project Lead [[phd/wiki/entities/romaric-nzoumbou-boko|Pr Romaric Nzoumbou-Boko]] (noted in coordination logistics as [[phd/wiki/entities/romaric|Romaric]]), Project Manager [[phd/wiki/entities/aboubacar-soumah|Dr Aboubacar Soumah]] ([[phd/wiki/entities/souma|Souma]]), [[phd/wiki/entities/didier-menard|Pr Didier Ménard]], and [[phd/wiki/entities/jean-de-dieu-longo|Pr Jean-de-Dieu Longo]]. The diagnostic workflow was designed by [[phd/wiki/entities/patrice-piola|Dr Patrice Piola]], with [[phd/wiki/entities/mehdi-el-alaoua|Mehdi El Alaoua]] serving as the primary developer and designer. Field deployment records ([[phd/wiki/sources/notes2026oadlog|OAD(log) - Beta-Bangui]]) outline a hardware strategy utilizing tablets budgeted at ~300€ each, which require explicit theft mitigation. The implementation timeline targeted equipping these tablets for internal testing in Bangui by June 21st, ahead of a version 1.2.0 public launch in September to integrate beta feedback and ultimately scale toward mid-October logistical targets.
